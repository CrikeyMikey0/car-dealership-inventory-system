# STRICT TDD RULES

Every feature must follow the exact workflow below.

---

## Step 1 — RED

Write the failing test first.

Run the tests.

Confirm the test fails.

No production code may exist before the failing test.

---

## Step 2 — GREEN

Write only enough implementation to satisfy the failing test.

Do not add extra functionality.

Run all tests.

Confirm they pass.

---

## Step 3 — REFACTOR

Improve the implementation.

Remove duplication.

Improve readability.

Improve component composition.

Keep every test passing.

---

Repeat this cycle for every component.

Never build multiple components before writing tests.
