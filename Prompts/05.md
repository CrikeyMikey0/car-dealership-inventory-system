# STRICT TDD RULES

Every feature must follow this workflow.

## Step 1 — RED

Write the failing test.

Run the tests.

Verify they fail.

No production code may exist before the failing test.

---

## Step 2 — GREEN

Write only enough production code to satisfy the failing test.

Nothing more.

Run all tests.

Verify they pass.

---

## Step 3 — REFACTOR

Improve the implementation.

Remove duplication.

Improve naming.

Improve readability.

Apply SOLID principles.

Run all tests again.

---

Repeat this cycle for **every endpoint**.

Never implement multiple endpoints before writing tests.
